# Sprint 57 — نظام الشخصيات الثماني

## الهدف

إدخال الشخصيات الثماني التي زوّد بها المستخدم إلى تطبيق سهيل، مع فصل خيارات الأولاد عن البنات بحسب نوع الحساب، واستخدام قصّات مناسبة لكل موضع بدل عرض الصورة الكاملة في جميع الصفحات.

## ما تم تنفيذه

- إضافة 4 شخصيات للأولاد و4 شخصيات للبنات بأصول WebP محسّنة وخلفية شفافة.
- إضافة اختيار **نوع الحساب: ولد / بنت** في التسجيل وفي إعداد رحلة الطالب.
- إذا كان الحساب ولدًا تظهر شخصيات الأولاد الأربع فقط.
- إذا كان الحساب بنتًا تظهر شخصيات البنات الأربع فقط.
- عند تغيير نوع الحساب تُلغى أي شخصية غير متوافقة ويُطلب اختيار شخصية من المجموعة الجديدة.
- حفظ `gender` و`avatarId` في الملف المحلي وفي API.
- منع API من حفظ شخصية لا تطابق نوع الحساب.
- عرض الشخصية المختارة في الصفحة الرئيسية، صفحة الحساب، وتجربة التحديات.
- إبقاء معرفات الشخصيات ثابتة: `male_01..04` و`female_01..04`.

## القصّات المعتمدة

لكل شخصية أربع نسخ محسّنة:

- `card` بمقاس 320×420 لاختيار الشخصية.
- `avatar` بمقاس 240×240 للرأس والكتفين في الحساب والرئيسية.
- `half` بمقاس 420×520 للتلميحات والبطاقات المستقبلية.
- `full` بمقاس 640×900 للاستخدامات التي تحتاج جسمًا كاملًا.

الواجهة لا تحمل الصور الأصلية الكبيرة؛ أصول التشغيل للشخصيات الثماني بجميع النسخ تقارب 0.84MB فقط.

## الشخصيات

### الأولاد

- سلمان — مختبر العلوم.
- راكان — ثوب وتوجيه.
- ياسر — طالب جامعي.
- فهد — ثوب وترحيب.

### البنات

- نورة — تركواز وتلميحات.
- لينا — وردي وترحيب.
- ريم — بنفسجي ومهام.
- جود — تحديات وإنجاز.

## التحقق

- اختيار الولد يعرض 4 شخصيات أولاد فقط.
- اختيار البنت يعرض 4 شخصيات بنات فقط.
- اختيار `female_03` حُفظ وعُرض في الحساب.
- التسجيل كحساب بنت نقل النوع إلى جلسة الدخول وصفحة الإعداد.
- API قبل ملف بنت مع شخصية بنت ورفض شخصية ولد لنفس الحساب.
- فحص Python وJavaScript نجح دون أخطاء.

## ملاحظات مستقبلية

- هذه الشخصيات أصبحت أصولًا رسمية للحساب، ويمكن لاحقًا إنشاء تعبيرات إضافية لنفس الشخصية للتلميحات والنجاح والستريك دون تغيير معرفها الأساسي.
- أسماء الشخصيات قابلة للتعديل من كتالوج `data/avatars/avatars.json` دون تغيير منطق الربط.
