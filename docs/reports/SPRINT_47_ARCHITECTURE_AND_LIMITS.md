# Sprint 47 — البنية والحدود

## ما أصبح منظمًا

- واجهة Sprint 47 مستقلة في ملفي CSS وJavaScript.
- الإعدادات والقوالب والنماذج في ملفات JSON منفصلة.
- الشخصيات أصول SVG مستقلة.
- API يوفر إعدادات عامة وتشخيصًا حسب الهدف والمسار.
- مخطط SQLite يحتوي على جداول تمهيدية للملفات الشخصية والصداقات والتحديات.

## ما لا يزال محليًا

- حسابات الطلاب محفوظة في المتصفح في نسخة Streamlit الحالية.
- إعدادات الأدمن في الواجهة تحفظ محليًا، بينما مسار API يحتاج توكن ومتغير بيئة.
- الصداقات والتحديات في الواجهة نموذج محلي، وليست متعددة الأجهزة بعد.
- الإشعارات عند إغلاق التطبيق تحتاج تطبيق Expo وخدمة Push.

## حدود التوسع

هذه الحزمة ليست مطالبة ولا مهيأة وحدها لتحمل 100 ألف مستخدم متزامن. قبل الإنتاج يلزم:

1. تطبيق React Native/Expo مستقل.
2. API خلف Load Balancer.
3. PostgreSQL مُدارة مع نسخ احتياطي وRead Replicas عند الحاجة.
4. Redis للكاش والجلسات ومعدلات الطلب.
5. تخزين أصول عبر CDN.
6. تسجيل دخول آمن وتشفير كلمات المرور وإدارة صلاحيات.
7. Queue للإشعارات والمهام غير المتزامنة.
8. مراقبة أخطاء، Logs، Metrics واختبارات تحميل.

## ملاحظة الدرجات

نموذج توقع سهيل داخلي وإرشادي. لا يمثل معادلة رسمية لهيئة تقويم التعليم والتدريب، ولا ينبغي عرضه على أنه ضمان لدرجة الاختبار.

## ملاحظة Streamlit

تم اختبار الحزمة على Streamlit 1.58.0 دون أخطاء تنفيذ عبر `AppTest`. ظهر تنبيه إهمال مستقبلي لـ `components.v1.html`، لذلك جرى تثبيت Streamlit تحت 1.59 مؤقتًا. هذا لا يؤثر على التشغيل الحالي، لكنه سبب إضافي لعدم اعتبار Streamlit واجهة المتجر النهائية والاستمرار إلى Expo.
