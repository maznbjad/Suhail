# Sprint 54 — الفحص البصري الآلي

تم توليد HTML الفعلي من `app.py` وتشغيله داخل Chromium Headless ثم اختبار الحالات التالية:

- شاشة الدخول ظاهرة دون قائمة سفلية.
- الانتقال إلى التطبيق يعرض الرئيسية النهائية.
- نار الستريك SVG موجودة ومرئية.
- القائمة النهائية مرئية في الرئيسية.
- القوائم القديمة مخفية.
- حساب الطالب يقبل القدرات والتحصيلي معًا.
- اختيار أدبي يظل خاصًا بالقدرات.
- صفحة تحديد المستوى توضح وجود قياسين مستقلين.
- بدء التحصيلي يعرض أسئلة تحصيلي فقط.
- أثناء القياس تختفي القائمة الرئيسية ولوحة الرفع.
- بعد الخروج من القياس تعود القائمة.
- داخل تفاصيل الملخص تختفي القائمة الرئيسية ويظهر مقبض الرفع.
- إذا حاول كود قديم استبدال الرئيسية، يعيد الحارس واجهة Sprint 54.
- لم تظهر أخطاء JavaScript أثناء السيناريو.

النتيجة التفصيلية موجودة في `SPRINT_54_VISUAL_QA.json`.
