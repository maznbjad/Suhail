# تقرير Sprint 89 — الهوية والحساب واليوزر

## قاعدة البناء

بُني هذا الإصدار على **Sprint 88** مع الحفاظ على الخطة الذكية، التحفيز، ملخصات الفيزياء، بنك الأسئلة وروابط الفيزياء.

## التعديلات المنفذة

### 1. شعار سهيل
- استُبدل الشعار القديم الذي يحتوي على الأبراج المالية بالصورة الجاهزة المرسلة من المستخدم.
- تم تحديث أصول التشغيل الثلاثة:
  - `assets/brand/suhail_logo_current.png`
  - `assets/images/suhail_logo_full_original.png`
  - `assets/runtime/suhail_logo.webp`
- تم استبدال الشعار المضمّن داخل ملف الافتتاحية المتحركة أيضًا، حتى لا يعود الشعار القديم من نسخة مخزنة داخل SVG.

### 2. إصلاح اختلاط Admin بحساب الطالب
- أصبحت صفحة الحساب تقرأ الاسم من جلسة المستخدم الحالي أولًا.
- أي ملف شخصي قديم يحمل اسم `Admin` لا يستطيع تجاوز اسم حساب الطالب الحالي.
- تتم مزامنة الاسم واليوزر والبريد المالك داخل ملف الطالب المحلي.
- حساب الطالب لا يرى أدوات الإدارة إلا إذا كانت صلاحية الجلسة `admin` فعليًا.

### 3. اليوزر الفريد
- إضافة خانة يوزر إلزامية عند إنشاء الحساب.
- الصيغة المقبولة: 3–20 حرفًا إنجليزيًا أو رقمًا مع السماح بالشرطة السفلية `_`.
- منع تكرار اليوزر محليًا وفي قاعدة بيانات API.
- ترقية آمنة للحسابات القديمة بإنشاء يوزر فريد مشتق من البريد عند الحاجة.
- يظهر اليوزر في صفحة الحساب وصفحة الأصدقاء، مع ضبط اتجاهه من اليسار إلى اليمين.

### 4. الأصدقاء عن طريق اليوزر
- البحث عن الصديق أصبح باليوزر دون الحاجة إلى إدخال كود الصداقة.
- يمنع النظام إضافة الحساب لنفسه أو إضافة الصديق نفسه مرتين.
- مسار API يقبل `receiver_username` ويحوّله إلى الحساب الصحيح.
- أكواد الصداقة القديمة بقيت داخليًا للتوافق مع التحديات، لكنها لم تعد العنصر الرئيسي الظاهر للطالب.

### 5. تنظيم صفحة الحساب
تم تقسيم الصفحة إلى مجموعتين منفصلتين:

- **الحساب والنشاط:** الرحلة التعليمية، تحديد المستوى، الوضع الداكن، الإشعارات، الأصدقاء والتحديات، والأسئلة المحفوظة.
- **عن سهيل:** عن سهيل، التواصل، الخصوصية، الشروط، والأسئلة الشائعة.

### 6. الشخصية
- يستخدم الحساب والرئيسية معرّف الشخصية نفسه من ملف الطالب.
- تم فرض `object-fit: contain` والتمركز من الأسفل بدل القص القسري.
- لا يتم تكبير الشخصية بطريقة تقطع الرأس أو اليد أو الكتاب داخل مواضع الحساب والرئيسية والتحفيز.

### 7. رؤوس قوائم الملخصات
- حذف الأيقونة الزخرفية الموجودة في الجهة اليسرى.
- حذف عبارة «كل القوائم بنفس التصميم وطريقة الاستخدام».
- حذف عبارة «نفس البطاقة في جميع المستويات».
- إضافة تنظيف احتياطي يمنع ظهور العبارات من أي نسخة محلية قديمة.

## نتائج التحقق

- فحص Sprint 89 الآلي: ناجح بالكامل.
- فحص Python: ناجح لـ `app.py` وAPI ومستودع الحسابات.
- فحص JavaScript: ناجح لجميع الملفات المعدلة.
- اختبار المتصفح المصغر:
  - ظهر الحساب باسم `f` بدل `Admin`.
  - ظهر اليوزر `@f_student`.
  - تطابقت الشخصية بين الرئيسية والحساب.
  - اختفت أيقونة رأس الملخص والعبارات الداخلية.
  - أخطاء JavaScript: صفر.
- تشغيل Streamlit: استجابة HTTP 200.
- اختبار API بقاعدة مؤقتة:
  - إنشاء الحساب `f_student`: HTTP 201.
  - قراءة المستخدم الحالي: HTTP 200.
  - إنشاء مستخدم ثانٍ `g_student`: HTTP 201.
  - إرسال طلب صداقة باليوزر: HTTP 201 وحالة `pending`.
- بنك الأسئلة محفوظ: **5,400 سؤال**.
- ملخصات الفيزياء محفوظة: **72 ملخصًا**.
- روابط أسئلة الفيزياء الدقيقة محفوظة: **600 رابط**.
