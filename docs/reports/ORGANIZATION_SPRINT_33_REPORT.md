# Sprint 33 — تنظيم الملفات قبل التطوير الإضافي

## الهدف
تنظيف جذر المشروع وترتيب الملفات قبل إضافة مزايا جديدة وقبل مسار تجهيز Apple Store.

## ما تم
- إبقاء ملفات التشغيل الأساسية فقط في الجذر.
- نقل تقارير السبرنتات الحديثة إلى `docs/reports/recent_sprints/`.
- نقل ملاحظات إصلاح التشغيل إلى `docs/setup/`.
- أرشفة `README.txt` القديم داخل `docs/archive/root_notes/`.
- إضافة `PROJECT_STRUCTURE.md` كخريطة رسمية للمجلدات.
- إضافة `docs/setup/RUN_WINDOWS.md` لتوضيح التشغيل.

## ما لم يتم عمداً
- لم يتم تفكيك `app.py` حتى لا يتأثر التشغيل الحالي.
- لم يتم حذف ملفات البيانات المكررة داخل `data/` لأن بعض التقارير والنسخ السابقة تشير إليها، ولأن الحذف الآن قد يسبب كسر غير ضروري.
- لم يتم تحويل البيانات إلى قاعدة بيانات؛ هذا سبرنت لاحق.

## توصية السبرنت التالي
تفكيك `app.py` تدريجيًا إلى:
- `src/config.py`
- `src/assets.py`
- `src/data_loader.py`
- `src/ui/`
- `src/pages/`
