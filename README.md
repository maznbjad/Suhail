# سهيل — Sprint 59

هذه الحزمة تجمع التطوير حتى Sprint 59 فوق النسخة السابقة، وتركز على تثبيت الواجهة، فصل تحديد مستوى القدرات عن التحصيلي، تحسين الستريك، تعميق الأدمن، تأسيس حسابات آمنة وصداقات وتحديات عبر API، ثم الفحص والأداء والأمان.


## Sprint 59 — إصلاح الملخصات والتنقل

- بوابة الملخصات تعرض ثلاثة مسارات فقط وبالترتيب: **تحصيلي، قدرات لفظي، قدرات كمي**.
- داخل التحصيلي تظهر المواد الأربع فقط: **فيزياء، كيمياء، رياضيات، أحياء**.
- لا تختلط مواد التحصيلي مع أقسام القدرات، ولا تظهر مهارات القدرات داخل التحصيلي.
- إجمالي المحتوى الظاهر حاليًا: **80 ملخصًا**؛ 72 تحصيليًا و8 للقدرات.
- القائمة السفلية موحدة وتبقى ظاهرة في جميع صفحات التطبيق بعد تسجيل الدخول، بما فيها تفاصيل الملخصات، وتختفي فقط أثناء الإجابة الفعلية عن أسئلة الاختبار أو قياس المستوى.
- شاشة إعداد الاختبار ليست اختبارًا فعليًا، لذلك تبقى القائمة ظاهرة فيها.
- إلغاء لوحة الرفع القديمة داخل الملخصات لتجنب وجود قائمتين.
- توحيد أيقونة العودة في الجهة اليمنى مع الحفاظ على وظيفة الرجوع الخاصة بكل مسار.
- بحث الملخصات يبدأ من حرفين ويصفي النتائج دون إعادة بناء الصفحة مع كل ضغطة.
- مسار الفيزياء الغني محفوظ: كتاب ← وحدة ← ملخص تفصيلي وقوانين.

## التصحيح المعتمد للمسارات

- الطالب يستطيع دراسة **القدرات والتحصيلي معًا**، أو اختيار أحدهما.
- **علمي/أدبي يخص القدرات فقط**:
  - قدرات علمي: 6 كمي + 6 لفظي في القياس الافتراضي.
  - قدرات أدبي: 4 كمي + 8 لفظي في القياس الافتراضي.
- **التحصيلي قياس مستقل وموحد**: رياضيات، فيزياء، كيمياء، وأحياء، ولا يتغير بسبب اختيار علمي أو أدبي في القدرات.
- لا يوجد اختبار تحديد مستوى مختلط. توجد نتيجتان وسجلان مستقلان: `qudrat` و`tahsili`.



## Sprint 56 — الوضع الداكن الشامل

- مفتاح الوضع الداكن في صفحة الحساب أصبح يغيّر مظهر **التطبيق كاملًا** فورًا.
- يشمل الرئيسية، المراجعة، المهام، الملخصات، تفاصيل الملخص، تحديد المستوى، إعداد الاختبار، الأسئلة، النتائج، الحساب، التحديات، الصفحات المحفوظة، وصفحات الأدمن.
- القائمة السفلية الموحدة وتفاصيل الملخصات تتبع الوضع الداكن.
- حفظ الاختيار في `localStorage` وتطبيقه مبكرًا قبل رسم الصفحات لتقليل وميض اللون الفاتح.
- تحديث لون واجهة النظام عبر `theme-color`، ودعم عناصر الإدخال والحالات الفارغة والبطاقات القديمة والجديدة.
- لا تُعكس الصور أو تتغير ألوان الرسومات؛ التغيير يقتصر على الخلفيات والأسطح والنصوص والحدود وعناصر التحكم.

## Sprint 55 — صفحة الحساب الجديدة

- إعادة تصميم صفحة الحساب وفق التقسيم المرجعي المعتمد: بطاقة موحدة كبيرة، صفوف واضحة، فواصل رفيعة، أيقونات خطية، وأسهم تنقل موحدة.
- الإبقاء على ملخص الحساب والشخصية والمسارات أعلى الصفحة دون ازدحام.
- إضافة وصول مباشر إلى رحلتي التعليمية، تحديد المستوى، الوضع الداكن، الإشعارات، الأصدقاء والتحديات، الأسئلة المحفوظة، عن سهيل، التواصل، الخصوصية، الشروط، والأسئلة الشائعة.
- الوضع الداكن يعمل مباشرة من نفس الصف.
- صفحات داخلية فعلية للإشعارات، التواصل، عن سهيل، الخصوصية، الشروط والأسئلة الشائعة.
- لوحة الأدمن وإدارة المحتوى تظهران في مجموعة مستقلة للأدمن فقط.
- عرض الإصدار وحقوق سهيل وزر تسجيل الخروج في نهاية الصفحة.

## التشغيل السريع على ويندوز

شغّل:

```text
run_suhail.bat
```

أو يدويًا:

```bash
pip install -r requirements.txt
streamlit run app.py
```

لتشغيل API المحلي:

```text
run_api.bat
```

ثم افتح `http://127.0.0.1:5000/health` للتأكد من عمله. نشر إعدادات الأدمن يحتاج ضبط متغير البيئة `SUHAIL_ADMIN_TOKEN` قبل تشغيل API.

## API الأساسي

- `GET /health`
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`
- `GET/PUT /api/v1/profile`
- `GET /api/v1/diagnostic?path=qudrat&track=literary` — يتطلب حسابًا ويعيد جلسة مؤقتة دون الإجابات الصحيحة.
- `GET /api/v1/diagnostic?path=tahsili` — يتطلب حسابًا ويتجاهل المسار الأكاديمي عمدًا.
- `POST /api/v1/diagnostic/<session_id>/submit` — تصحيح على الخادم وحفظ النتيجة.
- `GET /api/v1/diagnostic-results` — يعيد سجل القدرات والتحصيلي منفصلًا.
- `GET /api/v1/questions?test_format=محوسب&limit=50&offset=0` — يتطلب حسابًا، يدعم فصل المحوسب/الورقي، ويخفي الإجابات الصحيحة.
- `POST /api/v1/questions/grade` — تصحيح التدريب على الخادم.
- `POST/GET /api/v1/friend-requests`
- `POST /api/v1/friend-requests/<id>/respond`
- `GET /api/v1/friends`
- `POST/GET /api/v1/challenges`
- `GET /api/v1/challenges/<id>` — يعيد أسئلة التحدي دون الإجابات بعد تفعيله.
- `POST /api/v1/challenges/<id>/accept`
- `POST /api/v1/challenges/<id>/submit` — تصحيح ونتيجة مشتركة على الخادم.
- `GET/PUT /api/v1/admin/settings` مع `SUHAIL_ADMIN_TOKEN`
