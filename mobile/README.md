# Suhail Mobile — React Native + Expo

هذا المجلد هو أساس نسخة iPhone الأصلية القادمة، ولا يحوّل واجهة Streamlit الحالية إلى تطبيق متجر بصورة مباشرة.

## المسار المعتمد

- React Native + Expo من ويندوز.
- EAS Build وEAS Submit للبناء والتوقيع والرفع إلى TestFlight/App Store.
- Python API + PostgreSQL للخلفية بدل ملفات JSON المحلية في الإنتاج.
- الأصول المشتركة من `assets/` بعد ضغطها وتجهيز مقاسات iOS.

## الإشعارات

المجلد `notifications/` يحتوي:

- `suhailNotifications.ts` — أساس طلب الإذن، Push Token، التذكير اليومي، ومراجعات الاستدعاء المتباعد.
- `notification_plan.json` — أنواع الرسائل، الساعات الهادئة، وحدود التكرار.

هذه الملفات **أساس تنفيذي** للنسخة الأصلية، لكنها لا تجعل إشعارات الجوال المغلق تعمل من نسخة Streamlit. التشغيل الفعلي يحتاج مشروع Expo، إعداد EAS/APNs، وربط التوكن بخادم الإشعارات.

## ما ينتقل من Sprint 54 إلى Expo

- ملف الطالب: يمكنه اختيار القدرات والتحصيلي معًا؛ علمي/أدبي يخص القدرات فقط، إضافة إلى الشخصية.
- القائمة السفلية وقواعد إخفائها داخل الملخص والاختبار.
- قياسان مستقلان: قدرات حسب المسار، وتحصيلي موحد لجميع الطلاب، مع سجلين منفصلين.
- الستريك والساعة الرملية.
- صفحة الأصدقاء والتحديات.
- 8 شخصيات SVG من `assets/avatars/`.
- إعدادات عامة من `GET /api/v1/config/public`.


## عقد التنقل المعتمد

- الصفحات الرئيسية: القائمة السفلية ثابتة وتحترم Safe Area.
- تفاصيل الملخص: القائمة مخفية ويظهر مقبض لرفعها.
- الأسئلة والاختبارات: لا توجد قائمة سفلية.
- الدخول والافتتاحية: لا توجد قائمة سفلية.

## ما لا يزال مطلوبًا

- إنشاء مشروع Expo فعلي وشاشاته الأصلية.
- ربط تسجيل الدخول والملف والقياسات والستريك بالأطراف الموجودة في API.
- تسجيل Expo Push Token وتخزينه في الخادم وربطه بـ APNs.
- نقل SQLite إلى PostgreSQL واستخدام Redis للكاش والمهام عند الإنتاج.
